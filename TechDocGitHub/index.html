Introduction

Git is a distributed version control system that helps developers track changes in code, collaborate with others, and manage software projects efficiently.

GitHub is a cloud-based platform that hosts Git repositories, making it easier to collaborate with teams and contribute to open-source projects.

To use Git, you need to install it on your system. See the next section for installation instructions.

Getting Started with Git

Before using Git, you need to install it on your system:

  • Windows: Download and install Git from git-scm.com.
  • Mac: Install Git using Homebrew: brew install git.
  • Linux: Install Git via the package manager: sudo apt install git (Debian/Ubuntu) or sudo dnf install git (Fedora).

Once installed, verify Git is working by running:

git --version

This should return the installed version of Git.

Basic Git Commands

Here are some fundamental Git commands:

  • `git init` - Initializes a new Git repository.
  • `git add` - Stages changes for commit.
  • `git commit -m "message"` - Saves changes with a message.
# Initializing a Git repository git init
Branching and Merging

Branches allow you to work on different features independently.

  • Create a new branch with `git branch feature-branch`.
  • Switch to a branch with `git checkout feature-branch`.
  • Merge branches with `git merge feature-branch`.
# Creating a new branch git branch feature-branch
Working with GitHub

GitHub allows developers to store and collaborate on repositories online.

  • `git push` - Uploads local commits to GitHub.
  • `git pull` - Fetches updates from a remote repository.
# Pushing changes to GitHub git push origin main
Advanced Git Features

Git provides powerful features that help developers manage complex projects efficiently. Some of the most useful advanced commands include rebasing, stashing, cherry-picking, and Git hooks.

Rebasing

Rebasing allows you to integrate changes from one branch into another by rewriting commit history. Unlike merging, which creates a new commit, rebasing moves commits on top of the latest branch.

# Rebase your feature branch onto the main branch git checkout feature-branch git rebase main

If conflicts occur, resolve them, then continue with:

git rebase --continue

Stashing

Stashing allows you to temporarily save changes without committing them, so you can switch branches or pull updates without losing work.

# Stash your changes git stash

To list stashed changes:

git stash list

To apply the most recent stash:

git stash apply

To remove a stash after applying it:

git stash drop

Cherry-Picking

Cherry-picking allows you to apply specific commits from one branch to another.

# Cherry-pick a specific commit by its hash git cherry-pick 3a5b6c7

Use this when you want to apply only a particular fix or feature without merging an entire branch.

Git Hooks

Git Hooks are custom scripts that run automatically before or after Git commands like commits, merges, and pushes.

# Create a pre-commit hook (Linux/macOS) nano .git/hooks/pre-commit

Add a script inside the file (e.g., prevent commits with 'WIP' in the message):

#!/bin/sh if git log -1 --pretty=%B | grep -i "WIP"; then echo "Error: You cannot commit with 'WIP' in the message." exit 1 fi

Save the file and make it executable:

chmod +x .git/hooks/pre-commit

Now, this script will prevent commits with "WIP" in the message.

Squashing Commits

Squashing allows you to combine multiple commits into one before merging.

# Interactive rebase to squash commits git rebase -i HEAD~3

Replace "pick" with "squash" for the commits you want to combine, then save and exit.

Managing Remote Repositories

Sometimes, you may need to change the remote URL or work with multiple remotes.

# Change the remote repository URL git remote set-url origin https://github.com/your-username/new-repo.git

To add a new remote repository:

git remote add upstream https://github.com/original-repo.git
